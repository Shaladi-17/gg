## 🔍 شرح الكود سطر بسطر

```php

```
**⬆️ تجهيز الاستعلام** — يقول للسيرفر: "جهّز هذا الاستعلام لكن **لا تنفذه الآن**". نستخدم `prepare` بدل `query` لأن فيه متغير `?` جاي من المستخدم (أكثر أماناً).

---

```php
$rows->execute(array($nat));
```
**⬆️ التنفيذ** — الآن نفّذ الاستعلام! وضع قيمة `$nat` مكان علامة الاستفهام `?` في جملة SQL. لاحظ إنها **مصفوفة** `array(...)` حتى لو فيها قيمة واحدة.

> مثال: لو `$nat = "Libya"` والاستعلام `SELECT * FROM info WHERE nation=?`
> يصير الاستعلام الفعلي: `SELECT * FROM info WHERE nation='Libya'`

---

```php
$n = $rows->rowCount();
```
**⬆️ عدّ النتائج** — كم سجل رجعلي من قاعدة البيانات؟ يخزّن العدد في `$n`.

---

```php
if ($n > 0) {
```
**⬆️ فحص** — هل في نتائج؟ لو `$n = 0` معناه ما لقى شيء فما يدخل الحلقة.

---

```php
while ($row = $rows->fetch(PDO::FETCH_OBJ)) {
```
**⬆️ جلب سطر بسطر** — كل مرة الحلقة تدور، يجلب **سجل واحد** ككائن (Object). لما تخلص السجلات يرجع `false` والحلقة تتوقف.

`FETCH_OBJ` = نوصل للبيانات بـ **سهم** `->`

```php
echo $row->id . ":" . $row->name;
```
**⬆️ الطباعة** — يطبع مثلاً: `1:Ahmed` ثم `2:Sara` ثم `3:Ali`...

## 📊 ملخص مرئي

```
prepare($sql)          ← جهّز الاستعلام
       ↓
execute(array($nat))   ← نفّذ + حط القيم مكان ?
       ↓
rowCount()             ← كم سجل رجع؟
       ↓
fetch(FETCH_OBJ)       ← هات سجل واحد ككائن
```

## 🎯 القاعدة الذهبية — سؤالين فقط اسألهم

لما تشوف **أي كود PDO**، اسأل نفسك سؤالين فقط:

### السؤال 1: "أنا **أقرأ** بيانات ولا **أغيّر** بيانات؟"

| أقرأ (SELECT) | أغيّر (INSERT / UPDATE / DELETE) |
|---|---|
| استخدم `query()` أو `prepare()` | استخدم `exec()` |
| **يرجّعلي صفوف** أقراها | **ما يرجّع صفوف** — بس ينفّذ |
| أحتاج `fetch()` بعده | ما أحتاج `fetch()` |

```php
// أقرأ ← prepare + fetch
$rows = $conn->prepare("SELECT * FROM info");
$rows->execute();
while ($row = $rows->fetch(...)) { ... }

// أغيّر ← exec وخلاص
$conn->exec("INSERT INTO info VALUES (1, 'Ali', 'male', 'Libya')");
$conn->exec("UPDATE info SET name='Sara' WHERE id=1");
$conn->exec("DELETE FROM info WHERE id=1");
```

### السؤال 2: "فيه **متغير من المستخدم** ولا لا؟"

| فيه متغير من المستخدم ✅ | ما فيه متغير ❌ |
|---|---|
| استخدم `prepare()` + `execute()` | استخدم `query()` أو `exec()` |
| حط `?` مكان المتغير | اكتب القيم مباشرة |

```php
// فيه متغير ← prepare (آمن)
$sql = "SELECT * FROM info WHERE nation=?";
$rows = $conn->prepare($sql);
$rows->execute(array($nat));     // $nat جاي من المستخدم

// ما فيه متغير ← query (مباشر)
$rows = $conn->query("SELECT * FROM info");
```

## 📋 خلاصة في صورة واحدة

```
                    ┌──────────────────┐
                    │   ابدأ بجملة SQL  │
                    └────────┬─────────┘
                             │
                    ┌────────▼─────────┐
                    │ SELECT ولا غيره؟  │
                    └────────┬─────────┘
                   /                    \
            SELECT                   INSERT/UPDATE/DELETE
              │                              │
     فيه متغير؟                      $conn->exec($sql)
      /        \                         خلاص! ✅
    نعم        لا
     │          │
  prepare()   query()
  execute()     │
     │          │
  ┌──▼──────────▼──┐
  │ fetch() بحلقة   │
  │ واقرأ سطر سطر  │
  └────────────────┘
```

## ⚠️ نقطة مهمة جداً للامتحان

```
setcookie() ← يكتب الكوكي
           ↓
           ⚠️ ما تقدر تقراها في نفس الصفحة!
           ↓
$_COOKIE   ← تقراها في الصفحة الجاية (الطلب الثاني)
```

يعني:
```php
setcookie('bg', '#ff0000');
echo $_COOKIE['bg'];        // ❌ ما يشتغل أول مرة!

// لازم المستخدم يفتح صفحة ثانية أو يعمل refresh
// عشان المتصفح يرسل الكوكي للسيرفر
```

### Planner Response

## 📦 رفع الملفات — فكّر فيها كـ"شحن طرد بالبريد"

تخيل إنك تبي **ترسل صورة بالبريد**:

```
1. تحط الصورة في ظرف            ← المستخدم يختار الملف
2. البريد يستلمها في مخزن مؤقت   ← السيرفر يحفظها في tmp
3. أنت تنقلها للمكان النهائي     ← move_uploaded_file()
```

---

## 🔍 شرح سطر بسطر

### أولاً: الفورم (لازم تحفظه)
```html
<form method="post" enctype="multipart/form-data">
    <input type="file" name="images[]" multiple>
    <input type="submit" value="ارفع">
</form>
```
> ⚠️ بدون `enctype="multipart/form-data"` **ما يشتغل الرفع أبداً!**
> `images[]` ← الأقواس تعني **ملفات متعددة**

---

### ثانياً: الكود سطر بسطر

```php
if (isset($_FILES['images']) && !empty($_FILES['images']['name'][0])) {
```
> **هل المستخدم رفع شيء فعلاً؟**
> - `isset(...)` ← هل الحقل موجود؟
> - `!empty(...['name'][0])` ← هل الملف الأول **مش فاضي**؟
> - `[0]` = أول ملف في المصفوفة

---

```php
    $uploadDir = 'uploads/';
```
> **وين أبي أخزّن الملفات؟** في مجلد اسمه `uploads/`

---

```php
    foreach ($_FILES['images']['name'] as $index => $origName) {
```
> **لفّ على كل ملف** واحد واحد
> - `$index` ← رقم الملف (0، 1، 2...)
> - `$origName` ← اسم الملف الأصلي (مثلاً `photo.jpg`)

### 🔑 ما الذي يحتويه `$_FILES`؟

```
$_FILES['images'] = [
    'name'     => ['photo.jpg', 'cat.png'],        ← أسماء الملفات
    'tmp_name' => ['/tmp/abc123', '/tmp/def456'],   ← مكانها المؤقت
    'error'    => [0, 0],                           ← 0 = ما فيه خطأ
    'size'     => [54321, 12345],                    ← الحجم بالبايت
    'type'     => ['image/jpeg', 'image/png']        ← نوع الملف
]
```

---

```php
        if ($_FILES['images']['error'][$index] == UPLOAD_ERR_OK) {
```
> **هل الملف رقم `$index` انرفع بدون مشاكل؟**
> `UPLOAD_ERR_OK` = `0` = ما فيه خطأ ✅

---

```php
            $tmpName = $_FILES['images']['tmp_name'][$index];
```
> **وين الملف الآن؟** في المخزن المؤقت (مثل `/tmp/abc123`)
> هذا المكان **مؤقت** — لو ما نقلته يضيع!

---

```php
            $ext = pathinfo($origName, PATHINFO_EXTENSION);
```
> **استخرج الامتداد** من الاسم الأصلي
> - `photo.jpg` → `$ext = "jpg"`
> - `cat.png` → `$ext = "png"`

---

```php
            $newName = 'art_' . $articleId . '_' . $index . '_' . time() . '.' . $ext;
```
> **اصنع اسم جديد فريد** عشان الملفات ما تتكرر
> مثال: `art_5_0_1720209600.jpg`
>
> | الجزء | المعنى |
> |-------|--------|
> | `art_` | بادئة ثابتة |
> | `$articleId` | رقم المقال (5) |
> | `$index` | رقم الملف (0) |
> | `time()` | الوقت بالثواني (رقم فريد) |
> | `$ext` | الامتداد (jpg) |

---

```php
            $targetPath = $uploadDir . $newName;
```
> **المسار النهائي الكامل**
> `uploads/` + `art_5_0_1720209600.jpg` = `uploads/art_5_0_1720209600.jpg`

---

```php
            if (move_uploaded_file($tmpName, $targetPath)) {
                echo "تم الرفع بنجاح";
            }
```
> **انقل الملف** من المؤقت `/tmp/abc123` إلى `uploads/art_5_0_1720209600.jpg`
> لو نجح ← اطبع رسالة نجاح ✅

## ✍️ الآن اكتبه بمتغيرات مختلفة — مثلاً رفع صور طلاب

غيّر بس **الأسماء** والفكرة **نفسها بالظبط**:

```html
<form method="post" enctype="multipart/form-data">
    <input type="file" name="photos[]" multiple>
    <input type="submit" value="ارفع">
</form>
```

```php
if (isset($_FILES['photos']) && !empty($_FILES['photos']['name'][0])) {
    $folder = 'student_photos/';

    foreach ($_FILES['photos']['name'] as $i => $fileName) {
        if ($_FILES['photos']['error'][$i] == UPLOAD_ERR_OK) {
            $temp       = $_FILES['photos']['tmp_name'][$i];
            $extension  = pathinfo($fileName, PATHINFO_EXTENSION);
            $saveName   = 'std_' . $studentId . '_' . $i . '_' . time() . '.' . $extension;
            $destination = $folder . $saveName;

            if (move_uploaded_file($temp, $destination)) {
                echo "تم رفع صورة الطالب بنجاح";
            }
        }
    }
}
```

### 🔑 شوف الفرق — **بس الأسماء تغيّرت**:

| الأصلي | الجديد | نفس الوظيفة |
|--------|--------|-------------|
| `images` | `photos` | اسم الحقل في الفورم |
| `$uploadDir` | `$folder` | مجلد الحفظ |
| `$index` | `$i` | رقم الملف |
| `$origName` | `$fileName` | الاسم الأصلي |
| `$tmpName` | `$temp` | المسار المؤقت |
| `$ext` | `$extension` | الامتداد |
| `$newName` | `$saveName` | الاسم الجديد |
| `$targetPath` | `$destination` | المسار النهائي |
| `art_` | `std_` | البادئة |
| `$articleId` | `$studentId` | رقم السجل |

> **القالب واحد — غيّر الأسماء وخلاص!** 🎯
