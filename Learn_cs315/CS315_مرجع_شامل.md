## 2. تسجيل الدخول (Login) + Session + الصلاحيات (Priv)

```php
<?php
session_start();
include_once("connection_pdo.php");

function display_login_form() {
    ?>
    <form method="post" action="">
        اسم المستخدم: <input type="text" name="u_name" required/><br/>
        كلمة السر: <input type="password" name="password" required/><br/>
        <input type="submit" name="login" value="دخول"/>
    </form>
    <?php
}

function check_login($conn) {
    $u_name = $_POST['u_name'];
    $pass   = $_POST['password'];

    try {
        $sql = "SELECT * FROM user_info WHERE u_name=:u AND Password=:p";
        $stmt = $conn->prepare($sql);
        $stmt->execute(array(':u' => $u_name, ':p' => $pass));
        $row = $stmt->fetch(PDO::FETCH_OBJ);

        if ($row) {
            // تخزين بيانات المستخدم في الـ session
            $_SESSION['logged_in'] = true;
            $_SESSION['u_no']      = $row->u_no;
            $_SESSION['u_name']    = $row->u_name;
            $_SESSION['priv']      = $row->Priv;

            // توجيه حسب الصلاحية
            if ($row->Priv == 1) {
                header("Location: admin_page.php");
            } else {
                header("Location: user_page.php");
            }
            exit();
        } else {
            echo "اسم المستخدم أو كلمة السر غير صحيحة";
            display_login_form();
        }
    } catch(PDOException $e) {
        echo "Error: " . $e->getMessage();
    }
}

if (!isset($_POST['login'])) {
    display_login_form();
} else {
    check_login($conn);
}
?>
```

### حماية أي صفحة تحتاج تسجيل دخول (ضعها بأول كل صفحة محمية)

## 3. Insert (إضافة سجل جديد) — النمط الأساسي

```php
<?php
include_once("connection_pdo.php");

function display_form() {
    ?>
    <form method="post" action="">
        السعر: <input type="text" name="price" required/><br/>
        الوصف: <textarea name="description" required></textarea><br/>
        <input type="submit" name="add" value="إضافة"/>
    </form>
    <?php
}

function insert($conn) {
    $price = $_POST['price'];
    $desc  = $_POST['description'];

    // input validation بسيط
    if (!is_numeric($price)) {
        echo "السعر يجب أن يكون رقم";
        display_form();
        return;
    }

    try {
        $sql = "INSERT INTO mosque_info (price, description) VALUES (:price, :desc)";
        $stmt = $conn->prepare($sql);
        $stmt->execute(array(':price' => $price, ':desc' => $desc));
        echo "تمت الإضافة بنجاح";
    } catch(PDOException $e) {
        echo "Error: " . $e->getMessage();
    }
}

if (!isset($_POST['add'])) {
    display_form();
} else {
    insert($conn);
}
?>
```
**نصيحة:** استخدام `prepare()` + `execute()` بـ placeholders (`:price`) أفضل وأأمن من التضمين المباشر للمتغير جوا الـ SQL string (يحميك من SQL Injection)، بس لو الدكتورة بتستخدم أسلوبها القديم (`$conn->exec($sql)` مع تضمين المتغيرات مباشرة) خليك عليه لأنه هو اللي متعودة عليه بالتصحيح.

## 5. Delete (حذف) مع تحقق منطقي قبل التنفيذ (Business Logic Check)

**السيناريو (زي نقطة 6 بالأسايمنت):** المستخدم يقدر يلغي حجزه المبدئي بس لو نفس اليوم مش محجوز نهائي.

```php
<?php
include_once("connection_pdo.php");
session_start();

function display_form($conn) {
    // نعرض حجوزات المستخدم الحالي فقط (من الـ session)
    $u_no = $_SESSION['u_no'];
    $sql = "SELECT * FROM resv_Info WHERE u_no=:u AND Type=0";
    $stmt = $conn->prepare($sql);
    $stmt->execute(array(':u' => $u_no));
    ?>
    <form method="post" action="">
    <?php while ($row = $stmt->fetch(PDO::FETCH_OBJ)) { ?>
        <input type="radio" name="r_no" value="<?php echo $row->r_no; ?>">
        حجز رقم <?php echo $row->r_no; ?> - تاريخ <?php echo $row->r_date; ?><br/>
    <?php } ?>
    <input type="submit" name="cancel" value="إلغاء الحجز"/>
    </form>
    <?php
}

function cancel_reservation($conn) {
    $r_no = $_POST['r_no'];

    try {
        // الخطوة 1: نجيب بيانات الحجز المطلوب إلغاؤه (التاريخ والصالة)
        $sql1 = "SELECT * FROM resv_Info WHERE r_no=:r";
        $stmt1 = $conn->prepare($sql1);
        $stmt1->execute(array(':r' => $r_no));
        $reservation = $stmt1->fetch(PDO::FETCH_OBJ);

        // الخطوة 2: نتحقق - هل نفس اليوم ونفس الصالة محجوزين نهائياً (Type=1)؟
        $sql2 = "SELECT * FROM resv_Info WHERE r_date=:d AND no=:n AND Type=1";
        $stmt2 = $conn->prepare($sql2);
        $stmt2->execute(array(':d' => $reservation->r_date, ':n' => $reservation->no));

        if ($stmt2->rowCount() > 0) {
            // فيه حجز نهائي بنفس اليوم -> ما نسمحش بالإلغاء
            echo "لا يمكن إلغاء هذا الحجز لأن اليوم محجوز بشكل نهائي";
        } else {
            // مفيش تعارض -> ننفذ الحذف
            $sql3 = "DELETE FROM resv_Info WHERE r_no=:r";
            $stmt3 = $conn->prepare($sql3);
            $stmt3->execute(array(':r' => $r_no));
            echo "تم إلغاء الحجز بنجاح";
        }
    } catch(PDOException $e) {
        echo "Error: " . $e->getMessage();
    }
}

if (!isset($_POST['cancel'])) {
    display_form($conn);
} else {
    cancel_reservation($conn);
}
?>
```
**الفكرة الذهبية هون:** أي عملية حذف/تعديل حساسة = **SELECT تحقق أول، بعدين نفذ العملية جوا else**.

## 10. Input Validation (التحقق من صحة البيانات)

### أ) بالجافاسكريبت (client-side) — قبل الإرسال
```html
<form onsubmit="return validateForm()" method="post">
    الاسم: <input type="text" id="name" name="name"/><br/>
    الهاتف: <input type="text" id="phone" name="phone"/><br/>
    <input type="submit" value="إرسال"/>
</form>

<script>
function validateForm() {
    var name = document.getElementById("name").value;
    var phone = document.getElementById("phone").value;

    if (name.trim() == "") {
        alert("الرجاء إدخال الاسم");
        return false;
    }
    if (!/^[0-9]{10}$/.test(phone)) {
        alert("رقم الهاتف يجب أن يكون 10 أرقام");
        return false;
    }
    return true;
}
</script>
```

### ب) بالـ PHP (server-side) — دايماً لازم كمان تتحقق بالسيرفر
```php

        echo "<p style='color:red'>$e</p>";
    }

    // البيانات صحيحة -> كمّل بالعملية
}
?>
```

## 11. بناء تقويم (Calendar) بتلوين شرطي — النقطة الأصعب بالأسايمنت

**الفكرة:** نعرض جدول أيام الشهر الحالي، وكل يوم نتحقق هل فيه حجز، ونلونه حسب نوعه.

```php
<?php
include_once("connection_pdo.php");

function show_calendar($conn, $hall_no) {
    $current_month = date('m');
    $current_year  = date('Y');
    $days_in_month = cal_days_in_month(CAL_GREGORIAN, $current_month, $current_year);

    // نجيب كل حجوزات هذا الشهر لهذه الصالة
    $sql = "SELECT r_date, Type FROM resv_Info WHERE MONTH(r_date)=:m AND YEAR(r_date)=:y AND no=:n";
    $stmt = $conn->prepare($sql);
    $stmt->execute(array(':m' => $current_month, ':y' => $current_year, ':n' => $hall_no));

    // نخزن الحجوزات في array عشان الوصول السريع: [التاريخ => النوع]
    $reservations = array();
    while ($row = $stmt->fetch(PDO::FETCH_OBJ)) {
        $day = (int) date('d', strtotime($row->r_date));
        $reservations[$day] = $row->Type;
    }

    echo "<table border='1'><tr>";
    for ($day = 1; $day <= $days_in_month; $day++) {

        // نحدد اللون حسب حالة اليوم
        if (isset($reservations[$day])) {
            if ($reservations[$day] == 1) {
                $color = "red";     // حجز مؤكد (نهائي)
            } else {
                $color = "yellow";  // حجز مبدئي
            }
        } else {
            $color = "white";       // يوم فاضي
        }

        echo "<td style='background-color:$color; width:30px; text-align:center;'>$day</td>";

        // كل 7 أيام ننزل لسطر جديد
        if ($day % 7 == 0) {
            echo "</tr><tr>";
        }
    }
    echo "</tr></table>";

    echo "<p><span style='background-color:yellow'>&nbsp;&nbsp;&nbsp;</span> حجز مبدئي &nbsp;
          <span style='background-color:red'>&nbsp;&nbsp;&nbsp;</span> حجز مؤكد</p>";
}

show_calendar($conn, 1); // رقم الصالة كمثال
?>
```
**النقاط الأساسية اللي لازم تفهمها بهاد الكود:**
- `cal_days_in_month()` بترجعلك عدد أيام الشهر (تتأقلم تلقائياً مع فبراير 28/29).
- بنستخدم array مؤقت (`$reservations`) عشان ما نسويش query داخل كل حلقة (أداء أفضل وأسهل تصحيح).
- `strtotime()` + `date('d', ...)` تحول أي تاريخ لرقم اليوم بس.
- المنطق الشرطي (`if/else`) هو أساس التلوين — ممكن الدكتورة تغير الألوان أو تزيد حالة ثالثة (زي يوم عطلة)، بس نفس الهيكل بيضل ينطبق.

## 12. ملخص - "خوارزمية" التفكير بأي سؤال بالفاينل

```
1. اقرأ السؤال → حدد اسم الجدول المعني
2. حدد نوع العملية: SELECT / INSERT / UPDATE / DELETE
3. حدد شروط WHERE (بالذات: هل فيه شرط على صلاحية/تاريخ/نوع/مستخدم معين؟)
4. هل محتاج تحقق قبل التنفيذ (business logic check)? → استخدم SELECT تحقق أول
5. اكتب:
   - function display_form()  → الفورم
   - function do_action($conn) → العملية + try/catch
   - if (!isset($_POST['btn'])) display_form(); else do_action($conn);
6. لا تنسى input validation (JS + PHP)
7. لو محتاج صلاحيات → session_start() + تحقق $_SESSION['priv']
```

## أخطاء شائعة تفوّت منها علامات
- نسيان `session_start()` بأول الصفحة (لازم يكون أول سطر تقريباً).
- نسيان `[]` بعد اسم الـ checkbox/multiple select.
- نسيان `try/catch` حول أي عملية PDO.
- استخدام `$_POST` بدون التحقق `isset()` الأول (بيطلع Warning).
- نسيان `exit()` بعد `header("Location: ...")`.
- استخدام أسماء جداول/أعمدة مختلفة عن الموجودة بالسؤال بالظبط (حساسة لحالة الأحرف أحياناً).
