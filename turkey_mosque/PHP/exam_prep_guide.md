## 1️⃣ الأسئلة المقارنة الجوهرية (أسئلة نظرية وعملية مضمونة 100%)

غالباً ما يبدأ الامتحان بأسئلة "قارن بين" أو أسئلة اختيار من متعدد حول الفروقات الجوهرية. إليك أهمها:

### 🔹 المقارنة الأولى: `include` مقابل `require` (مهم جداً)
* **المشترك:** كلاهما يستدعي ويضمن ملفاً خارجياً في الصفحة الحالية.
* **الاختلاف عند الفشل (عدم وجود الملف):**
  * **`include`**: تصدر **تحذيراً (Warning)** فقط ويستمر السكريبت في العمل بشكل طبيعي.
  * **`require`**: تصدر **خطأ فادحاً (Fatal Error)** وتوقف تنفيذ السكريبت فوراً.
* **اللاحقة `_once` (مثل `include_once` و `require_once`):**
  * تتأكد أولاً إذا كان الملف قد تم تضمينه مسبقاً في الصفحة؛ فإذا كان كذلك، تتجاهل التضمين الجديد لمنع إعادة تعريف الدوال أو المتغيرات وحدوث أخطاء.

---

### 🔹 المقارنة الثانية: علامات الاقتباس المفردة `''` مقابل المزدوجة `""`
* **المزدوجة `""` (Double Quotes):** مرنة جداً؛ تقوم بمحاكاة وقراءة المتغيرات وتفسير رموز الهروب (مثل `\n` للسطر الجديد و `\t` للمسافة).
* **المفردة `''` (Single Quotes):** سريعة وجامدة؛ تطبع النص حرفياً كما هو دون قراءة للمتغيرات.
* **مثال امتحاني لتتبع المخرج:**
  ```php
  $name = "أحمد";
  echo "مرحباً $name\n"; // المخرج: مرحباً أحمد (مع سطر جديد بالخلفية)
  echo 'مرحباً $name\n'; // المخرج: مرحباً $name\n (حرفياً)
  ```

---

### 🔹 المقارنة الثالثة: الدالة `print` مقابل الدالة `echo`
* **الدالة `echo`**: أداة لغة (Language Construct) لا ترجع أي قيمة (Void)، وهي أسرع وتدعم طباعة عدة سلاسل نصية مفصولة بفاصلة `,` (مثال: `echo "A", "B", "C";`).
* **الدالة `print`**: دالة حقيقية ترجع قيمة منطقية `1` دائماً عند نجاح الإخراج، مما يسمح باستخدامها بداخل التعبيرات المنطقية، وتطبع وسيطاً واحداً فقط.

---

### 🔹 المقارنة الرابعة: الأسلوب `GET` مقابل الأسلوب `POST` في النماذج
* **`GET`**: ترسل البيانات بشكل مكشوف عبر شريط العنوان (URL)، وتكون محدودة الحجم وتصلح لعمليات البحث والمشاركة.
* **`POST`**: ترسل البيانات بشكل مخفي داخل جسم طلب HTTP، وغير محدودة الحجم، وهي الأكثر أماناً لإرسال كلمات المرور والبيانات الحساسة ورفع الملفات.

## 2️⃣ أسئلة تتبع مخرجات الكود (Output Tracing)

الأساتذة يحبون اختبار فهمك لكيفية عمل المفسر. ركز على الحالات التالية:

### 💡 الحالة أ: المتغيرات الساكنة (Static Variables)
* **السؤال المتوقع:** ما هي مخرجات الكود التالي؟
```php
function keep_track() {
    STATIC $count = 0;
    $count++;
    echo $count . " ";
}
keep_track();
keep_track();
keep_track();
```
* **الإجابة:** `1 2 3 `
* **السبب:** لأن المتغير المعرّف بـ `STATIC` لا يضيع من الذاكرة عند الخروج من الدالة، بل يحتفظ بآخر قيمة له للاستدعاء القادم. (لو أزلنا كلمة STATIC لكان المخرج `1 1 1`).

---

### 💡 الحالة ب: التحويل النوعي التلقائي (Implicit Type Casting)
* **السؤال المتوقع:** ما ناتج العمليات التالية؟
```php
$sum = 12 + "15"; 
echo $sum; // المخرج: 27
// التفسير: يتم تحويل النص "15" تلقائياً لرقم وإضافته للـ 12.

$total = 10 + "45 fire engines";
echo $total; // المخرج: 55
// التفسير: بما أن النص يبدأ برقم، تقرأ PHP الرقم 45 وتهمل باقي النص.

$total2 = 10 + "fire engines 45";
echo $total2; // المخرج: 10
// التفسير: بما أن النص يبدأ بحروف وليس أرقام، يتم اعتباره صفراً (0).
```

---

### 💡 الحالة ج: التمرير بالإشارة (Pass by Reference)
* **السؤال المتوقع:** ما هي قيم المتغيرات بعد استدعاء الدالة؟
```php
$cost = 20;
$tax = 0.05;

function calculate_cost(&$cost, $tax) {
    $cost = $cost + ($cost * $tax);
    $tax += 4;
}
calculate_cost($cost, $tax);
echo "Cost: " . $cost . " | Tax: " . $tax;
```
* **الإجابة:** `Cost: 21 | Tax: 0.05`
* **التفسير:** المتغير `$cost` تم تمريره بالإشارة لوجود الرمز `&` في تعريف المعامل، فتأثر بالعملية داخل الدالة وتغير خارجها. أما `$tax` فتم تمريره بالقيمة العادية، وبالتالي لم تتغير قيمته خارج الدالة رغم زيادته بالداخل.

## 3️⃣ كتابة الأكواد الأساسية (Code Writing Templates)

غالباً سيطلب منك كتابة كود كامل للاتصال أو الإدراج أو الحذف. احفظ هذه القوالب الصخرية:

### 📝 قالب الاتصال بقاعدة البيانات (بـ PDO - كما في مجلدك `sre`):
```php
<?php
$dsn = "mysql:host=localhost;dbname=student;charset=utf8";
$username = "root";
$password = "";

try {
    $conn = new PDO($dsn, $username, $password);
    // تفعيل إلقاء الاستثناءات عند الأخطاء
    $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch(PDOException $e) {
    die("Connection failed: " . $e->getMessage());
}
?>
```

### 📝 قالب إضافة سجل آمن باستخدام PDO (Prepared Statements):
```php
// جلب البيانات من النموذج
$id = $_POST['id'];
$name = $_POST['name'];
$gender = $_POST['gender'];
$nation = $_POST['nation'];

try {
    // 1. صياغة الاستعلام مع علامات الاستفهام (?) للآمان
    $sql = "INSERT INTO info (id, name, gender, nation) VALUES (?, ?, ?, ?)";
    
    // 2. تجهيز الاستعلام
    $stmt = $conn->prepare($sql);
    
    // 3. التنفيذ وتمرير البيانات كمصفوفة مرتبة
    $stmt->execute([$id, $name, $gender, $nation]);
    
    echo "تم الحفظ بنجاح!";
} catch(PDOException $e) {
    echo "حدث خطأ: " . $e->getMessage();
}
```

### 📝 قالب حذف السجلات المحددة عبر Checkboxes (كما في `delete_pdo.php`):
```php
// استقبال مصفوفة المعرفات المحددة
$selected_ids = $_POST['id']; 

try {
    $sql = "DELETE FROM info WHERE id = ?";
    $stmt = $conn->prepare($sql);
    
    foreach ($selected_ids as $val) {
        $stmt->execute([$val]);
    }
    echo "تم حذف السجلات المحددة بنجاح!";
} catch(PDOException $e) {
    echo "Error: " . $e->getMessage();
}
```

## 4️⃣ التحقق والفلترة (Validation & Forms)

### 🔹 كيف تستقبل Checkboxes متعددة بنفس الاسم؟
في كود HTML، يجب كتابة الاسم مع أقواس مصفوفة:
```html
<input type="checkbox" name="id[]" value="1"> طالب 1
<input type="checkbox" name="id[]" value="2"> طالب 2
```
وفي خادم PHP، نستقبلها كمصفوفة ونمر عليها بـ `foreach` أو نتحقق من وجود قيمة معينة بـ `in_array()`:
```php
$selected = $_POST['id']; // مصفوفة
if (in_array("1", $selected)) {
    // الطالب الأول تم تحديده
}
```

### 🔹 فلترة البيانات بـ `filter_var_array()`:
دالة تفحص مصفوفة كاملة من المدخلات دفعة واحدة بناءً على قواعد فلترة مسبقة:
```php
$filters = array(
    "age"   => array("filter"  => FILTER_VALIDATE_INT,
                      "options" => array("min_range" => 20, "max_range" => 55)),
    "email"  => FILTER_VALIDATE_EMAIL
);
$valid = filter_var_array($_POST, $filters);
// إذا كانت القيمة غير صالحة، ستكون قيمتها بالمصفوفة الناتجة false
```

---

> [!TIP]
> **نصيحة للامتحان:**
> * تأكد دائماً من كتابة الفاصلة المنقوطة `;` في نهاية كل سطر برمجي بـ PHP.
> * تأكد من وضع رمز الـ `$` قبل أسماء المتغيرات دائماً.
> * عند استخدام `header('Location: ...')` تذكر إلحاقها بـ `exit();` لمنع استمرار تنفيذ الكود بالخلفية.
> * اكتب التعليقات بوضوح لتُظهر للمصحح أنك تفهم منطق الكود.
